## v2.0.0 核心变更：只处理未按规范命名的 Session

**关键逻辑：** 在遍历所有 session 时，先检查每个 session 的 title：

- 如果 title 匹配正则 `^[a-z][a-z_]+-\d{10}$` → **跳过，不改动**
- 如果 title 不匹配 → 按原流程处理（检查内容、分类、重命名）

格式说明：以小写字母开头 + 下划线 + 短横线 + 10位数字（YYMMDDHHMM）

包含的跳过场景：
- 已按之前 skill 自动命名的 session → 跳过
- 任何符合 `topic-YYMMDDHHMM` 格式的 title → 跳过

需要处理的场景：
- `—`（未命名）→ 检查内容后命名
- uuid 格式 ID 的 session → 检查内容后命名或删除
- 有随意标题但缺少规范时间戳的 → 重新按规范命名

## 命名格式

```
<topic_name>-YYMMDDHHMM
```

规则：短横线两侧无空格，年份2位，时间戳无下划线。总长度 ≤ 28 字符。

## 时间戳获取

`started_at` → `created_at` → messages timestamp。格式 `strftime('%y%m%d%H%M')`。

## 执行步骤

0. 获取 session 列表，逐个检查 title 是否匹配 `^[a-z][a-z_]+-\d{10}$`
   - 匹配 → 跳过（已完成）
   - 不匹配 → 继续
1. `hermes sessions list --limit 60` 查看
2. `hermes sessions export` 看内容（只对不匹配的 session）
3. 先删 cron session、单条测试 session、空 session
4. 按话题分组，同话题按时间排序
5. 计算 `len(f"{topic}-{ts}")`，超28缩短
6. `hermes sessions rename` 重命名
7. 验证
